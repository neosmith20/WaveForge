Test Checklist

1. Read from Radio.

2. Save Backup from that same fresh read.

3. Change either the boot text or the boot display mode. Do not combine them in one write yet.

4. Run either Write Boot Text Only or Write Boot Mode Only and wait for the immediate readback verify to pass.

5. Run Read Back & Verify after text changes, then save a fresh post-write backup if needed.

6. Power-cycle the radio and do another fresh read to confirm the setting persisted.

Protocol Notes

This page is the only place where the proven boot-text and boot-mode write paths are exposed for now. The main CPS page stays read-only until more write paths are hardware-proven.

Read path: open the port, clear only buffered accumulator state, then send direct R requests. No C 00 or C FE init commands are sent before reads.

Read responses: every response is treated as a strict R-framed packet. The reader sync-scans for 0x52, logs and discards any pre-sync bytes, then reads the big-endian payload length and exact payload.

Boot text write path on STM32: X 01 prepare the 4 KB sector containing 0x7540, X 02 overlay the 32 boot-text bytes, then X 03 erase/program the sector. X 04 must not be used on STM32 because it can ACK without modifying flash.

Boot display mode write path on STM32: X 01 prepare the 4 KB sector containing 0x7518, X 02 overlay exactly one byte at 0x7518, then X 03 erase/program the sector. Immediate readback must return 0x00 for Picture or 0x01 for Text.
